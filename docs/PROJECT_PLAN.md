# 다독 리뷰 (Dadoc Review) 프로젝트 계획서

본 문서는 '다독 리뷰' 프로젝트의 목표, 요구사항 및 개발 일정을 정리한 문서입니다.

## 1. 프로젝트 개요 및 목표
**프로젝트 명:** 다독 리뷰 (Dadoc Review)
**핵심 목표:** 단순한 리뷰 나열이 아닌, 외부 리뷰 데이터를 수집·분석하여 사용자에게 가치 있는 도서 통찰을 제공하고 실제 대출 가능한 도서관 정보를 연결하는 지능형 도서 플랫폼 구축.

### 주요 가치 (Value Proposition)
1.  **AI 기반 통찰:** 산재한 리뷰들을 AI가 요약하여 책의 핵심을 빠르게 파악하도록 도움.
2.  **실질적 접근성:** 책의 가치를 발견한 즉시 주변 도서관의 소장 여부를 확인하여 행동으로 연결.
3.  **신뢰성 및 법적 준수:** 크롤링 데이터의 저작권을 준수하며 가공된 정보를 제공.

---

## 2. 요구사항 정의 (Requirements)

### 2.1 기능적 요구사항 (Functional)
-   **도서 정보 수집:** 온라인 서점 및 리뷰 플랫폼에서의 데이터 크롤링 및 외부 API 연동.
-   **AI 리뷰 요약:** 수집된 리뷰 데이터를 바탕으로 장단점 및 핵심 요약 생성.
-   **도서관 소장 정보 조회:** 공공도서관 API를 연계하여 사용자 위치 기반 또는 지역 기반 소장/대출 여부 확인.
-   **도서 검색:** 제목, 저자, ISBN 등을 통한 도서 검색 기능.

### 2.2 기술적 요구사항 (Technical)
-   **Backend:** FastAPI (Python)를 활용한 고성능 비동기 API 서버.
-   **Database:** SQLite (초기) / SQLAlchemy (ORM) 활용.
-   **AI 연동:** Gemini API 등을 활용한 텍스트 분석 및 요약.
-   **External APIs:** 공공도서관 정보나루, 온라인 서점 API 등.
